# <a name="install-script"></a>Yükleme betiği

PowerShell komut dosyalarını çevrimiçi havuzların yerel bilgisayara yükler.

## <a name="description"></a>Açıklama

Yükleme betiği cmdlet bulur ve bir veya daha fazla komut dosyası bir çevrimiçi galeriden indirir, doğrular ve bunları belirtilen yükleme kapsam için yerel bilgisayarda yükler.

Kapsam tanımlandığında veya kapsam parametresinin değeri AllUsers olduğunda betik %systemdrive%:\Program Files\WindowsPowerShell\scripts yüklenir. Currentuser'a kapsam değeri betik $home\Documents\WindowsPowerShell\scripts yüklenir.

Belirtilen komut dosyalarını en az ve tam sürümlerine dayalı sonuçlarınızı filtreleyebilirsiniz.

Bazı önemli notlar:
- Komut dosyaları yüklü tek bir dosyalarıdır. Sonuç olarak, bir komut dosyası yalnızca bir kopyasını yüklenir ve komut dosyaları birden çok sürümünün yüklü yan yana bir sistemde olamaz. 
- Komut dosyaları, yükleme betiğini çalıştırdığınızda, yüklenecek dış modülleri üzerinde bağımlılıkları tanımlayabilir.
- **Komut isteminde güvenilmeyen:** kullanıcı kabul güvenilmeyen bir depodan komut dosyalarını yüklemek için gereklidir.
- RequiredVersion, PowerShell sürüm 5.0 veya daha yeni olan sürümleriyle SxS içinde belirtilen sürümü yükler.

Joker karakter kullanılması desteklenmez, - yükleme betiği, adına Kaydet-komut dosyası ve kaldırma komut dosyası cmdlet'leri.

### <a name="scope"></a>Kapsam
Komut dosyası yükleme kapsamını belirtir. Bu parametre için kabul edilebilir değerler: AllUsers ve Currentuser'a.

Varsayılan yükleme AllUsers kapsamıdır.

Diğer bir deyişle, bilgisayarın tüm kullanıcıları için erişilebilen bir konumda yüklenmesi komut dosyalarının AllUsers kapsam sağlar "$env: SystemDrive\Program Files\WindowsPowerShell\scripts".

Komut dosyası yalnızca geçerli kullanıcı için kullanılabilir olmasını sağlamak Currentuser'a kapsam komut dosyalarının yalnızca "$home\Documents\WindowsPowerShell\scripts için", yüklenmesini sağlar.


Komut dosyası yükleme kapsamını belirtir. Geçerli değerler: AllUsers ve Currentuser'a. Currentuser'a varsayılandır.

Komut dosyası tüm kullanıcılar tarafından kullanılabilir olmasını sağlamak için %systemdrive%:\ProgramFiles\WindowsPowerShell\Scripts bir komut dosyası yüklemek için AllUsers kapsamı belirler. Komut dosyası yalnızca geçerli kullanıcı için kullanılabilir olmasını sağlamak $home\Documents\WindowsPowerShell\Scripts komut dosyasını yüklemek için Currentuser'a kapsamını belirtir.


## <a name="nopathupdate"></a>NoPathUpdate

- NoPathUpdate anahtar parametresini yükleme betiği cmdlet komut dosyası yükleme konumu PATH ortam değişkenine eklemek için istemi atlar.
- Herhangi bir kullanımından komutu – belirtilen NoPathUpdate ile hiçbir istemi ve yol güncelleştirilmekte değil neden olur (zorla yoksayılabilir burada).
- -YOLUN güncelleştirilir ve force – NoPathUpdate olmadan hiçbir isteminde neden olur.
- Ne – Force veya – NoPathUpdate belirttiyseniz, Kullanıcı istemi görürsünüz.
- Tüm bunlar yalnızca geçerlidir yükleme betiği verilen kapsam içinde ilk defa.


## <a name="notes"></a>Notlar

Bu cmdlet, Windows PowerShell 3.0 veya sonraki sürümleri Windows PowerShell, Windows 7 veya Windows 2008 R2 ve Windows'un sonraki sürümleri üzerinde çalışır.

Bir bilgisayarda komut dosyası adı parametresi için belirtilen değer eşleştiğinden ve MinimumVersion veya RequiredVersion parametresini eklemediniz yükleme betiği sessizce komut dosyaları yüklemeden devam eder. MinimumVersion veya RequiredVersion parametreler belirtildi ve var olan betiği Bu parametre değerleri eşleşmiyor, bir hata oluşur. Daha belirgin olması: şu anda yüklü komut dosyasının sürümünü MinimumVersion parametresinin değerini daha düşük veya RequiredVersion parametre değerine eşit değil ise, bir hata oluşur. Yüklü komut dosyasının sürümünü MinimumVersion parametresinin değerinden büyük veya RequiredVersion parametresinin değeri eşit ise, yükleme betiği komut dosyaları yüklemeden sessizce sürdürür.

Betik belirtilen adla eşleşen çevrimiçi galeriden varsa yükleme komut dosyası bir hata döndürür.

Birden fazla komut dosyası yüklemek için bir dizi virgülle ayırarak komut dosyası adını belirtin. Birden çok komut dosyası adı belirtirseniz, MinimumVersion veya RequiredVersion ekleyemezsiniz.

Varsayılan olarak, komut dosyaları Program Files klasörüne yüklenir. Yükleme komut dosyası birden çok PSGetItemInfo nesnelere iletebildiğiniz; Bu tek bir komutta yüklemek için birden fazla komut dosyası belirtmenin başka bir yoludur.

Yüklenen kötü amaçlı kod içeren çalışan komut dosyalarını engellemeye yardımcı olmak için komut dosyaları otomatik olarak yükleme tarafından içeri aktarılmadı. Güvenlik açısından en iyisi, herhangi bir cmdlet veya işlevleri, bir komut dosyası ilk defa çalıştırmadan önce bir kod değerlendirin.

## <a name="install-script-cmdlet-in-pipeline-operations"></a>Ardışık Düzen işlemlerinde yükleme betiği cmdlet'i

```powershell

# Find a script and install it
Find-Script -Name "MyDSC*" | Install-Script

# Find a script and install it to the CurrentUser scope
Find-Script -Name "MyDSC*" | Install-Script -Scope CurrentUser

# Find commands by name and install them
# The first command finds the specified commands in the INT repository, and then uses the pipeline operator to pass them to Install-Script to install them.
# The second command uses Get-Installedscript to verify the scripts from the prior command are installed.
Find-Command -Repository "INT" -Name Get-ContosoClient,Get-ContosoServer | Install-Script
Get-Installedscript

# Find multiple role capabilities and install them
Find-RoleCapability -Name MyJeaRole, Maintenance | Install-Script
Get-Installedscript

```

## <a name="install-script-and-get-installedscript-cmdlets"></a>Yükleme komut dosyası ve Get-InstalledScript cmdlet'leri
Yükleme betiği cmdlet'i, belirtilen kapsam için bir özel komut dosyası bağımlılıklarını birlikte yüklemek için olanak tanır. Varsayılan olarak, komut dosyaları AllUsers kapsama yüklenir. Get-InstalledScript cmdlet yükleme betiği cmdlet'i kullanılarak yüklenen komut dosyalarını listesini almak için olanak sağlar.

Kullanım Not: Yönetim ve komut dosyaları onlar yüklendikten sonra bulma izin vermek için yükleme betiği $home\Documents\WindowsPowerShell\Scripts adresindeki komut dosyaları depolamak için varsayılan klasörü oluşturur ve bu klasörü yolu ortamınıza eklemek. Yolun değiştirilmesi önemliyse, yükleme komut dosyası yerine Kaydet-komut dosyası kullanın. Yalnızca GET-InstalledScripts ve kaldırma komut dosyası yükleme komut dosyasını kullanarak sistemde yerleştirilmiş komut dosyaları ile çalışabilirsiniz.
```powershell
# Install locations for scripts:
# Default scope is AllUsers.
# AllUsers scope --> "$env:ProgramFiles\\WindowsPowerShell\\Scripts"
# CurrentUser scope -->; "$env:USERPROFILE\\Documents\\WindowsPowerShell\\Scripts"
```
